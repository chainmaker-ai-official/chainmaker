This document explains how the connection rendering system works between output and input ports in the Designer blocks.

1. Core Components:

ConnectionOverlay.jsx
- Main SVG overlay component positioned absolutely over the workspace
- Renders all connection lines using SVG path elements
- Handles both established connections and active "rubber band" wires
- Uses gradient strokes for visual styling

useFlowConnections.js
- Custom React hook managing connection state and logic
- Handles connection creation, deletion, and coordinate calculations
- Manages active wire state during connection creation
- Provides connection data to ConnectionOverlay

geometry.js
- Utility functions for SVG path creation
- createBezierCurve() generates smooth, sharp, or straight line paths
- Uses cubic Bezier curves for flowing connections

2. Connection Flow:

Step 1: Connection Start
- User clicks the output port (magenta bottom bar) in DraggableBlock.jsx
- onMouseDown handler calls onConnectionStart with block ID and port type
- This triggers the connection creation process

Step 2: Active Wire Creation
- useFlowConnections hook creates an activeWire object with:
  * sourceId and sourcePort (output)
  * startX, startY (port center coordinates)
  * currentX, currentY (updated during mouse movement)
- Mousemove listener updates current position as user drags

Step 3: Port Detection
- As user drags, system detects hover over valid input ports:
  * Checks for input port color (teal, green, or cyan)
  * Checks for data-input-port attribute
  * Adds visual hover feedback
- Uses getPortPosition() to calculate exact port coordinates

Step 4: Connection Completion
- On mouseup, system checks if valid target port is found
- Calculates target port coordinates using getPortPosition()
- Creates connection object with:
  * sourceId, targetId
  * sourcePort: 'output', targetPort: 'input'
  * sourceOffset, targetOffset (relative to block positions)
- Dispatches to Redux store and updates local state

3. Line Rendering:

ConnectionOverlay.jsx renders SVG paths:
- For each connection, calculates source and target port positions
- Uses createBezierCurve() to generate smooth path data
- Applies gradient stroke styling
- Updates dynamically when blocks move

4. Dynamic Updates:

- When blocks are dragged, useEffect in ConnectionOverlay forces re-render
- getPortPosition() recalculates port coordinates from DOM
- Connections automatically update to stay connected to correct ports
- Active wire updates in real-time during connection creation

5. Key Functions:

getPortPosition(blockId, portType, workspaceRef)
- Queries DOM for block element
- Finds port element (bottom bar for output, top bar for input)
- Calculates absolute coordinates relative to workspace
- Returns {x, y} coordinates or null if not found

createBezierCurve(startX, startY, endX, endY, style)
- Generates SVG path data for different curve styles
- Smooth: horizontal control points for flowing look
- Sharp: orthogonal step curve
- Straight: simple linear path
- Returns path data string for SVG d attribute

6. Visual Features:

- Gradient strokes (blue to purple for connections, yellow to orange for active wire)
- Smooth cubic Bezier curves for natural flow
- Visual hover feedback during connection creation
- Real-time updates during block dragging
- Pointer-events: none on SVG to allow clicks through to blocks

7. Integration Points:

DraggableBlock.jsx
- Output port has onMouseDown handler for connection start
- Visual styling indicates connection capability
- Passes connection events up to parent components

Workspace.jsx
- Renders ConnectionOverlay component
- Passes connection data and block positions
- Manages overall workspace layout

This system provides a smooth, intuitive connection experience where users can visually connect blocks by dragging from output to input ports, with lines automatically updating as blocks are moved around the workspace.
