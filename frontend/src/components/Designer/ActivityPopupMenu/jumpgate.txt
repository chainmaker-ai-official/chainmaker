=== ACTIVITYPOPUPMENU FUNCTIONALITY ===

src/components/Designer/ActivityPopupMenu/ActivityPopupMenu.jsx
  └── Main container ("Brain") - manages popup visibility, handles click on menu items, dispatches addActivity to Redux, calls onAddToWorkspace to add blocks to canvas

src/components/Designer/ActivityPopupMenu/utils/menuData.js
  └── Defines categories (System, Data Provider, Processing, Logic, Execution, Risk, Notifications) and activitiesByCategory mapping; provides useMenuData and useFilteredMenuData hooks

src/components/Designer/ActivityPopupMenu/utils/activityHelpers.js
  └── getActivityByMenuItemId() - looks up activity from marketplaceActivities or structure.json; isActivityAdded() - checks if activity already exists in palette

src/components/Designer/ActivityPopupMenu/hooks/useAnimation.js
  └── useAnimation() - manages popup animation states (hidden→entering→visible→exiting); useEscapeKey() - closes popup on Escape key press

src/components/Designer/ActivityPopupMenu/components/MenuHeader.jsx
  └── Displays header with added activity count and close button

src/components/Designer/ActivityPopupMenu/components/MenuSearch.jsx
  └── Search input for filtering activities by name

src/components/Designer/ActivityPopupMenu/components/MenuItem.jsx
  └── Renders primary menu item (category or activity); shows hover/selected states and "Added" badge

src/components/Designer/ActivityPopupMenu/components/SubMenu.jsx
  └── Renders nested submenu when category is hovered/selected

=== FLOW SUMMARY ===
1. User opens popup → useAnimation handles fade-in
2. User browses/searches activities → useFilteredMenuData filters by searchQuery
3. User clicks activity → handlePrimaryItemClick checks if already added
4. If not added → dispatch(addActivity(activity)) + onAddToWorkspace() adds block to canvas
5. Ctrl+Click → directly adds to workspace without adding to palette
6. Escape key → useEscapeKey closes popup
