=== DESIGNER COMPONENT STRUCTURE ===

src/components/Designer/Designer.jsx
  └── Main orchestrator - combines all components, manages state, coordinates between Toolbar, Workspace, ActivityPopupMenu, and Chat

src/components/Designer/Toolbar.jsx
  └── Top toolbar with File/Edit/View menus and Run button; provides placeholder handlers for save/export/undo/redo/etc.

src/components/Designer/hooks/useBlockManager.js
  └── Manages blocks state - addBlock(), deleteBlock(), updateBlockPosition(); dispatches removeConnectionsForBlock to Redux when deleting

src/components/Designer/hooks/useBlockDrag.js
  └── Handles block dragging - startDrag(), handleDrop(), handlePaletteDrop(); calculates positions relative to workspace

src/components/Designer/hooks/useFlowConnections.js
  └── See "src/components/Designer/Blocks/jumpgate.txt" for drag line connectivity - startConnection(), activeWire, dispatch(addConnection)

src/components/Designer/components/Workspace.jsx
  └── See "src/components/Designer/Blocks/jumpgate.txt" - container for blocks and ConnectionOverlay

src/components/Designer/components/BlockRenderer.jsx
  └── See "src/components/Designer/Blocks/jumpgate.txt" - renders appropriate block type based on block.id/category

src/components/Designer/components/ConnectionOverlay.jsx
  └── See "src/components/Designer/Blocks/jumpgate.txt" - SVG bezier curves between blocks

src/components/Designer/ActivityPopupMenu/
  └── See "src/components/Designer/ActivityPopupMenu/jumpgate.txt" - popup menu for adding activities

=== FLOW SUMMARY ===
1. Designer initializes useBlockManager, useFlowConnections, useBlockDrag hooks
2. User clicks in workspace (Ctrl+Click) → ActivityPopupMenu opens
3. User selects activity → handleAddToWorkspace → blockManager.addBlock adds to workspace
4. User drags block → useBlockDrag.startDrag updates position via useBlockManager
5. User connects blocks → see "src/components/Designer/Blocks/jumpgate.txt" for drag line process
6. Toolbar provides File/Edit/View actions (mostly placeholders)
