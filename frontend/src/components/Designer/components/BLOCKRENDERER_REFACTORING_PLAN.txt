/**
 * BlockRenderer Refactoring Plan
 * ==============================
 * 
 * PROBLEM DESCRIPTION
 * -------------------
 * 
 * Currently, BlockRenderer.jsx uses explicit if/else conditions to determine 
 * which component to render for each block type:
 * 
 *   if (isClockTimer) { return <SystemClock ... /> }
 *   if (isLiveOrderBook) { return <LiveOrderBookDraggableBox ... /> }
 *   if (isDataView) { return <DataView ... /> }
 *   if (isBang) { return <Bang ... /> }
 *   if (isLog) { return <Log ... /> }
 *   if (isUniswapRouter) { return <UniswapRouter ... /> }
 *   return <DraggableBlock ... />
 * 
 * Issues with this approach:
 * 
 * 1. VIOLATES OPEN/CLOSED PRINCIPLE
 *    - Must modify BlockRenderer every time a new block is added
 *    - Each new block requires 3 changes: import, detection, render
 * 
 * 2. CODE DUPLICATION
 *    - The same props are passed to each block (id, label, type, position, etc.)
 *    - Large blocks of repeated JSX for common props
 * 
 * 3. DETECTION LOGIC SPRREAD
 *    - Each block has multiple detection strategies (id, title, component, UUID)
 *    - Detection is hardcoded in the component, not configurable
 * 
 * 4. MAINTENANCE BURDEN
 *    - Hard to add new blocks
 *    - Easy to forget to update BlockRenderer
 *    - No single source of truth for block type mappings
 * 
 * 
 * PROPOSED SOLUTION
 * -----------------
 * 
 * Option 1: Component Map Pattern (Recommended)
 * 
 * Create a mapping from block types to components, and use it to dynamically
 * resolve the correct component:
 * 
 *   // Component map - single source of truth
 *   const BLOCK_COMPONENTS = {
 *     'SystemClock': SystemClock,
 *     'LiveOrderBookDraggableBox': LiveOrderBookDraggableBox,
 *     'DataView': DataView,
 *     'Bang': Bang,
 *     'Log': Log,
 *     'UniswapRouter': UniswapRouter,
 *     // New blocks only need to be added here!
 *   };
 * 
 *   // Detection + Resolution
 *   const getBlockComponent = (block) => {
 *     // Priority: component property > auto-detection
 *     if (block.component && BLOCK_COMPONENTS[block.component]) {
 *       return BLOCK_COMPONENTS[block.component];
 *     }
 *     return DraggableBlock; // Default
 *   };
 * 
 *   // Render
 *   const BlockComponent = getBlockComponent(block);
 *   return <BlockComponent {...commonProps} />;
 * 
 * 
 * Option 2: Auto-discovery with Dynamic Imports
 * 
 * Use the component name to dynamically import:
 * 
 *   const BlockComponent = React.lazy(() => 
 *     import(`../Blocks/${block.component}`)
 *   );
 * 
 * Trade-offs:
 * - Pros: Truly auto-discoverable, no map to maintain
 * - Cons: Harder to debug, may have bundler issues, no fallback control
 * 
 * 
 * SPECIAL HANDLING
 * ---------------
 * 
 * Some blocks need special props or callbacks that aren't shared:
 * 
 * - Bang: needs `onBang` callback with Redux dispatch
 * - DataView: needs `data` prop with mock/real data
 * - SystemClock: needs `onTick` callback
 * 
 * Solution: Use a configuration object for special props:
 * 
 *   const BLOCK_SPECIAL_PROPS = {
 *     'Bang': (block, props, dispatch) => ({
 *       ...props,
 *       onBang: (id, isOn) => { /* Bang-specific logic */ }
 *     }),
 *     'DataView': (block, props) => ({
 *       ...props,
 *       data: generateMockData(block)
 *     }),
 *     'SystemClock': (block, props) => ({
 *       ...props,
 *       onTick: (value) => { clockTickData[block.id] = value; }
 *     })
 *   };
 * 
 * 
 * IMPLEMENTATION STEPS
 * -------------------
 * 
 * 1. Create a new file: blockRegistry.js
 *    - Export BLOCK_COMPONENTS map
 *    - Export BLOCK_SPECIAL_PROPS configuration
 *    - Export detection utilities
 * 
 * 2. Refactor BlockRenderer.jsx
 *    - Import from blockRegistry
 *    - Replace if/else chain with component resolution
 *    - Use special props configuration
 * 
 * 3. Update structure.json
 *    - Add 'component' property to each trading_module
 *    - e.g., component: 'UniswapRouter'
 * 
 * 4. Benefits after refactoring:
 *    - Adding new blocks only requires:
 *      a. Create the block component in Blocks/
 *      b. Add entry to blockRegistry.js
 *      c. Add 'component' to structure.json
 *    - No more BlockRenderer modifications for new blocks!
 *    - Detection logic is centralized and configurable
 * 
 * 
 * EXAMPLE: Adding a new block after refactoring
 * ------------------------------------------------
 * 
 * Before (current approach):
 * 1. Create UniswapRouter.jsx
 * 2. Add import to BlockRenderer
 * 3. Add isUniswapRouter detection
 * 4. Add render condition
 * 5. Add to structure.json
 * 6. Add to constants.js activityIdMap
 * 
 * After (refactored):
 * 1. Create UniswapRouter.jsx
 * 2. Add to blockRegistry.js: 'UniswapRouter': UniswapRouter
 * 3. Add 'component': 'UniswapRouter' to structure.json
 * 
 * That's it! BlockRenderer never needs to change again.
 * 
 */
